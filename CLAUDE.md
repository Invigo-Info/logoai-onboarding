## Project Overview

Logo.ai prelaunch landing page built with **Next.js 14 (App Router)**, **TypeScript**, and **React 18**. It features a countdown timer, email waitlist signup via Supabase, and marketing sections. Launch date: March 20, 2026.

## Commands

```bash
npm install          # Install dependencies
npm run dev          # Dev server at http://localhost:3000
npm run build        # Production build — run before PRs to catch type/route errors
npm start            # Serve production build locally
```

No test runner, linter, or formatter is configured.

## Architecture

### Content-driven single-page app

All marketing copy lives in **uppercase config objects** at the top of `app/page.tsx` (`HERO`, `WHAT_YOU_GET`, `HOW_IT_WORKS`, `USE_EVERYWHERE`, `WHO_ITS_FOR`, `WHY_JOIN`, `FAQ_ITEMS`, `FINAL_CTA`, `FOOTER`). The JSX below renders these objects. Edit copy there — don't hard-code text in JSX.

### Key files

- **`app/page.tsx`** (~1,076 lines) — Entire page: content config objects, internal components (`Countdown`, `SignupForm`, `FAQSection`, `RevealSection`), inline SVG icons
- **`app/layout.tsx`** — Root layout, 7 Google Fonts loaded via `next/font/google` and CSS variables
- **`app/globals.css`** — Full design system using CSS custom properties (no Tailwind). Dark theme with accent colors (`--accent: #4df0c8`, `--accent-2: #6c63ff`, `--accent-3: #ff6b9d`)
- **`app/api/waitlist/route.ts`** — POST (email signup) and GET (waitlist count) endpoints using Supabase service role key

### Backend

The waitlist API uses `@supabase/supabase-js` with a server-only service role key. The Supabase `waitlist` table schema:
```sql
create table public.waitlist (
  id bigint generated by default as identity primary key,
  email text unique not null,
  source text default 'landing-page',
  created_at timestamptz default timezone('utc', now())
);
```

### Environment variables (`.env.local`)

- `NEXT_PUBLIC_SUPABASE_URL` — Public Supabase project URL
- `SUPABASE_SERVICE_ROLE_KEY` — Server-only secret; never prefix with `NEXT_PUBLIC_`

## Code Conventions

- TypeScript React function components, `"use client"` directive on page.tsx
- Two-space indentation, double quotes, trailing commas in multi-line objects
- `const` for component-level data, camelCase variables, PascalCase components
- CSS custom properties in `globals.css`; no Tailwind or CSS modules currently
- New UI elements go in `components/` and get imported into the page
- Commit style: imperative, scope-prefixed (e.g., `feat: refresh hero gradient`, `fix: gate countdown timer`), first line under 72 chars
